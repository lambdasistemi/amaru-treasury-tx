# Data Model: Local Devnet Smoke

## LocalDevnetRun

Represents one isolated smoke execution.

**Fields**

- `runId`: stable identifier for this run.
- `runDir`: absolute directory containing generated artifacts.
- `phase`: one of `node`, `withdraw`, `disburse`, `all`.
- `startedAt`: UTC start timestamp.
- `completedAt`: UTC completion timestamp when available.
- `status`: `starting`, `node-ready`, `waiting-rewards`,
  `building`, `passed`, `failed`, or `interrupted`.
- `socketPath`: node socket path once available.
- `nodeLogPath`: copied or linked node log path.
- `summaryPath`: machine-readable and human-readable summary path.

**Validation**

- `runDir` must be created fresh or explicitly emptied before use.
- A successful run must include a summary and node readiness evidence.
- Stale artifacts from prior runs must not be reused.

**State transitions**

`starting -> node-ready -> waiting-rewards -> building -> passed`

Any state may transition to `failed` or `interrupted` with a recorded
diagnostic.

## DevnetTiming

Records the effective short-epoch timing used for withdrawal smoke.

**Fields**

- `networkName`: expected to be `devnet`.
- `networkMagic`: expected to be `42`.
- `epochLengthSlots`: configured epoch length from genesis.
- `slotLengthSeconds`: configured slot length from genesis.
- `epochDurationSeconds`: derived epoch duration.
- `systemStart`: system start used by the running devnet.
- `observedTip`: current tip after readiness.

**Validation**

- `networkMagic` must equal `42`.
- `epochDurationSeconds` must be small enough for the configured
  reward wait budget.
- Timing evidence must be recorded before treasury actions run.

## PreparedTreasuryState

Represents local chain state required by release-facing Amaru flows.

**Fields**

- `walletFundingTxIn`: live UTxO used for wallet-funded fees and
  collateral.
- `treasuryTxIn`: live treasury script UTxO when available.
- `registryReferenceTxIn`: live registry reference input when
  available.
- `permissionReferenceTxIn`: live permissions reference input when
  available.
- `rewardAccount`: reward account under observation.
- `seededBy`: harness step that created the state.

**Validation**

- Required UTxOs must be present at the node tip used by the smoke.
- Missing treasury or registry state must produce a specific
  missing-state diagnostic, not partial build artifacts.
- Signing/submission details belong only to harness setup metadata.

## WithdrawalObservation

Captures the reward-account evidence before generating a withdraw
intent.

**Fields**

- `rewardAccount`: account queried from live chain state.
- `lastRewardLovelace`: latest observed reward amount.
- `firstPositiveAt`: UTC timestamp when rewards first became positive.
- `lastTip`: tip at the last reward observation.
- `lastEpoch`: epoch at the last reward observation when available.
- `waitBudgetSeconds`: configured maximum wait time.

**Validation**

- Successful withdrawal smoke requires `lastRewardLovelace > 0`.
- A timeout failure must include the last observed reward value, tip,
  and wait budget.

## SmokeArtifactSet

Files generated by one smoke run.

**Fields**

- `timingPath`: JSON timing evidence.
- `withdrawIntentPath`: withdrawal intent JSON when the withdrawal
  phase passes.
- `disburseIntentPath`: disburse intent JSON when the disburse phase
  passes.
- `buildLogPath`: `tx-build` transcript.
- `unsignedCborPath`: unsigned transaction CBOR output.
- `reportJsonPath`: machine-readable report.
- `reportMarkdownPath`: rendered report when requested.

**Validation**

- Every successful phase must print the relevant artifact paths.
- `tx-build` artifacts must come from live local-chain queries, not
  checked-in fixtures.
- No artifact may be reported as successful unless it was written
  during the current `LocalDevnetRun`.
