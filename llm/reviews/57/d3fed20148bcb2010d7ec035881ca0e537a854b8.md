- `docs/withdraw.md:88` — the JSON example shows `"treasuryRewardAccount": "stake_test1..."` but the schema (9d69ac5) defines that field as `$ref: hex28` (28-byte hex) and the synthetic fixture uses `"treasuryRewardAccount": "32201dc1e82708364c6c42a53f89f675314bb9ad5da2734aa10baa0d"`. A bech32 stake address would fail schema validation. An operator copying this snippet by hand would hit a confusing schema error. Replace the placeholder with the hex form, e.g.:

  ```json
  "withdraw": {
    "treasuryRewardAccount": "32201dc1e82708364c6c42a53f89f675314bb9ad5da2734aa10baa0d",
    "rewardsLovelace": 12345678
  }
  ```

  Optionally add a one-line note that the field is the 28-byte treasury stake-script hash hex, not a bech32 stake address — that's the kind of thing the schema enforces but a human reader won't infer from the field name alone.
