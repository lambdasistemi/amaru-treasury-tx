- `lib/Amaru/Treasury/Backend/N2C.hs:127` — `Left _mismatch -> error "queryStakeRewardsLovelace: era mismatch — node not in Conway"` raises a partial `error`. This is the same anti-pattern we just removed at `Main.hs:980` (replaced with `abortWithdraw`). An era-mismatch on an unsynced node or during an era transition will now manifest as an uncaught exception with no typed `WweAborted` trace, breaking the "every failure is a typed line" contract operators are scripted against. Two ways to fix:
  1. Change the return to `IO (Either StakeRewardsError Integer)` (or similar) and have the caller (`providerToWithdrawResolverEnv`) translate `Left` into `abortWithdraw tr "..."`.
  2. Wrap the call site in `try @SomeException` and emit `abortWithdraw` for the exception case.

  Option 1 is cleaner because it makes the era-mismatch a typed value rather than an exception.
