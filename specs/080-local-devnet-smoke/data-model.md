# Data Model: DevNet Governance Action Slice

## LocalDevnetRun

Represents one isolated smoke execution.

**Fields**

- `runId`: stable identifier for this run.
- `runDir`: absolute directory containing generated artifacts.
- `phase`: one of `node`, `governance`, or `all`.
- `startedAt`: UTC start timestamp.
- `completedAt`: UTC completion timestamp when available.
- `status`: `starting`, `node-ready`, `preparing-governance`,
  `submitted`, `observed`, `passed`, `failed`, or `interrupted`.
- `socketPath`: node socket path once available.
- `nodeLogPath`: copied or linked node log path.
- `summaryPath`: machine-readable and human-readable summary path.

**Validation**

- `runDir` must be fresh for each run.
- A successful run must include node readiness evidence.
- Stale artifacts from prior runs must not be reused.

## DevnetTiming

Records the local chain timing used by governance and follow-up slices.

**Fields**

- `networkName`: expected to be `devnet`.
- `networkMagic`: expected to be `42`.
- `epochLengthSlots`: configured epoch length from genesis.
- `slotLengthSeconds`: configured slot length from genesis.
- `epochDurationSeconds`: derived epoch duration.
- `systemStart`: system start used by the running DevNet.
- `observedTip`: current tip after readiness.

**Validation**

- `networkMagic` must equal `42`.
- Timing evidence must be recorded before governance setup runs.
- The epoch duration must be short enough for manual DevNet experiments.

## GovernanceSetupState

Represents local chain state required before submitting the
treasury-withdrawal governance action.

**Fields**

- `fundingTxIn`: setup UTxO used for local fees/deposits.
- `treasuryScriptHash`: hash of the Amaru treasury script stake credential.
- `rewardAccount`: reward account receiving protocol treasury funds.
- `registrationCertificate`: certificate artifact or tx body reference.
- `voteDelegationCertificate`: always-abstain vote delegation artifact
  or tx body reference.
- `protocolTreasuryAmount`: available local treasury/reserve amount.
- `seededBy`: harness step that created the local setup state.

**Validation**

- The stake credential setup must match the original Amaru intent:
  registration plus always-abstain vote delegation.
- Missing setup funds or protocol treasury funds must fail before
  governance submission.
- Signing/submission details belong only to harness metadata.

## TreasuryWithdrawalGovernanceAction

Represents the Conway governance action submitted by the smoke.

**Fields**

- `txId`: transaction that submitted the action.
- `actionId`: governance action id.
- `rewardAccount`: script reward account targeted by the withdrawal.
- `amountLovelace`: amount requested from protocol treasury state.
- `anchorUrl`: optional action anchor URL.
- `anchorHash`: optional action anchor hash.
- `policyHash`: optional constitution/policy script hash when required.
- `submittedAtTip`: node tip at submission.
- `observedAtTip`: node tip when the action boundary was observed.

**Validation**

- `amountLovelace` must be positive.
- The target reward account must be a script credential account for
  the Amaru treasury script.
- Successful output must include both `txId` and `actionId`.

## GovernanceEvidenceSet

Files generated by a governance smoke run.

**Fields**

- `timingPath`: JSON timing evidence.
- `certificatesPath`: certificate/setup summary.
- `actionPath`: governance action summary.
- `summaryPath`: top-level summary JSON.
- `logPath`: human-readable transcript.
- `upstreamBlockers`: linked upstream issues or PRs when support is missing.

**Validation**

- Every successful governance run must print the relevant artifact paths.
- No artifact may be reported as successful unless it was written
  during the current `LocalDevnetRun`.
- Missing upstream support must be explicit and linked.
